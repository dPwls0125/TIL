# 왜 클로드 코드를 사용해야하는가?

**클로드 코드**는 앤트로픽의 최신 모델인 Claude Opus 4를 기반으로 하며 **터미널에서 실행하는 에이전티 도구**이다. 

커서 AI도 클로드 소넷과 오퍼스 모델을 사용하는데 무슨 차이가 있는가?
모델은 모델일 뿐, 해당 모델을 사용할때 에이전틱 워크플로를 제공하고 최적화하는 서비스는 다르다.
커서는 기존의 intellij와 같은 개발환경을 사용하던 사람들은 커서에 적응해야만한다. 그러나 클로드 코드는 터미널 기반이기 때문에 자신이 선호하는 통합 개발 환경은 유지하면서도 사용할 수 있다.  

=> 터미널에서 살아 숨 쉬는 에이전틱 코딩 도구! ㅋㅋ

기존 LLM의 경우에는 프로젝트의 맥락을 읽을 수 없고 말그대로 코드를 복붙해야했기 때문에 워크플로우가 파편화 되어있었다. 
클로드 코드는 프로젝트 구조 분석, 작업 계획 수립, 실행, 검증 및 개선의 과정까지 4단계 워크플로를 통한 프로젝트 개발을 지원한다. 

그리고 플래닝 모드를 제공한다고 하는데, 상세한 계획을 제작해서 사용자에게 제시하는 능력이 강력하다고 한다. 그러나 아직 안서봐서 모름. 

나는 Max는 가격 부담이 심해서 Pro로 (17dollars) 입문해볼 예정이다. 5시간에 45개의 메세지를 보낼 수 있다고 함. 근데 짧은 메세지 기준이라서 개발에 사용하려면 더 적게 느껴질 수 있다고 한다.

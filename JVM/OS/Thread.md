# Thread 

Definition : 실행의 단위, **프로세스를 구성하는 실행의 흐름 단위**

- 하나의 프로세스가 한번에 여러 일을 처리할 수 있게 됨. => 프로세스를 구성하는 여러 명령어를 동시헤 실행할 수 있게 된것.


- 프로세스의 스레드들은 실행에 필요한 **최소한의 정보(프로그램 카운터를 포함한 레지스터, 스택)만을 유지한 채 프로세스 자원을 공유하며 실행**된다. 

- 최근의 운영체제는 CPU에 처리할 작업을 전달할 때 프로세스가 아닌 스레드 단위로 전달한다.  
리눅스는 프로세스와 스레드 모두 실행의 문맥이라는 점에서 동일하다고 간주하여 다르게 구분하지 않음. 


여기서 질문!

"동일한 작업을 수행하는 단일 스레드 프로세스 여러 개를 실행하는 것과 하나의 프로세스를 여러 스레드로 실행하는 것은 무엇이 다를까요?"

내 생각 : **자원의 효율성** 측면에서 다르다. 단일 프로세스 여러개로 print를 실행한다고 했을 때 독립된 PCB를 가지며 코드, 데이터, 힙을 비롯한 영역을 독립적으로 가져야함. 그러나 쓰레드 단위로 실행하게 되면, 코드, 데이터, 힙 영역을 공유한 채로 최소한의 메모리를 할당하여 실행할 있다. 

각기 다른 쓰레드 ID, 프로그램 카운터 값을 포함한 레지스터, 스택을 가질 뿐임. 코드 데이터 힙 영역은 공유!

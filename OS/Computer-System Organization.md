
# Computer-System Organization (2026.03.18)

오늘날의 computer-system은 하나 이상의 **CPU**와 구성요소와 **공유 메모리** 사이의 액세스를 제공하는 **공통 버스를 통해 연결된** **여러 장치 컨트롤러로** 구성된다.

### 장치 컨트롤러 
- 하드웨어이다.
- 장치 컨트롤러는 장치와 로컬 버퍼 사이의 데이터 전송을 담당한다.
- 읽기 작업에서는 보통 장치에서 로컬 버퍼로 데이터가 들어오고,
- 쓰기 작업에서는 로컬 버퍼의 데이터가 장치로 전송된다.

### 장치 드라이버 
- 소프트웨어이다.
- 장치 드라이버는 장치 컨트롤러의 레지스터에 작업 명령과 제어에 필요한 정보(예: 주소, 크기, 상태)를 기록한다.

### 입출력 작업을 예시로 시뮬레이션을 해보자. 

1) 입출력 작업을 수행하기 위해, 장치 드라이버는 장치 컨트롤러의 레지스터에 명령을 적재한다.
2) 장치 컨트롤러가 해당 명령을 수행하고 로컬 버퍼에 데이터를 적재한다.
3) 장치 컨트롤러가 장치 드라이버에게 작업이 끝났음을 알린다. <--- 인터럽트  
4) 읽기 작업이었다면 장치 드라이버는 읽어 온 데이터를 운영체제의 다른 부분이 사용할 수 있도록 전달한다.
경우에 따라 데이터 자체가 전달되거나, 데이터가 저장된 위치에 대한 정보가 전달될 수 있다.  
쓰기 작업이었다면 장치 드라이버는 작업 완료 여부나 장치 상태 정보를 운영체제에 알린다.

### 컨트롤러는 장치 드라이버에게 작업을 완료했다는 사실을 어떻게 알리는가? (인터럽트 - 구동 방식의 입출력 사이클)

하드웨어는 어느 순간이든 시스템 버스를 통해 CPU에 신호를 보내 인터럽트를 발생시킬 수 있다. 

내가 이해한 바로는, 
1) 데이터 처리가 완료되면 **장치 컨트롤러가** 데이터 처리가 완료되었다는 **인터럽트를 발생시키고,**
2) CPU가 그것을 받고
3) OS/드라이버쪽 핸들러를 실행한다.

## CPU의 인터럽트 처리 방식 정리

1) CPU는 진행중인 작업의 명령어 실행이 끝날 때마다 interrupt line을 감지하여 interrupt 발생 여부를 감지한다.
2) 인터럽트가 발생하면 CPU는 현재 작업을 재개할 수 있도록 필요한 상태를 저장한 뒤, 인터럽트 핸들러로 제어를 넘긴다.
3) intterrupt 핸들러가 데이터 처리를 완료하면 제어권을 돌려놓고 기존 작업을 이어서 수행한다. (인터럽트로 중단된 작업 재개)

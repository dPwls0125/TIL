# network architecture 

- 네트워크 아키텍쳐 : 대화에 사용하는 언어 처럼, 네트워크시 사용하는 언어
  ex) TCP/IP, OSI, Microsoft NETBEUI, IBM SNA 등등. 
- 프로토콜 : 언어 안에, 문법, 문자의 표기, 관용 표현 등등 여러 규약이 있듯이 여러개의 프로토콜이 모여서 하나의 네트워크 아키텍쳐가 됨. 따라서 프로토콜은 통신에 필요한 규칙이다

**복수의 프로토콜을 조합하여 네트워크 아키텍쳐가 된다.**
- TCP / IP 네트워크 아키텍쳐로 예를 들면, TCP/IP는 네개의 계층으로 구분하여 프로토콜을 조합한다. 계층 구분의 장점은 원하는 프로토콜만 구분하여 수정이 가능하다는 점이 될 것이다.

  | TCP/IP 계층 | 주요 프로토콜 |
  | ----------| ---------|
  | 어플리케이션층| HTTP, SMTP, POP3, DHCP, DNS 등| 
  | 트랜스포트 층 |TCP / UDP | 
  | 인터넷 층 | IP, ICMP, ARP 등 |
  | 네트워크 인터페이스 층| 이더넷, 무선 LAN, PPP 등|

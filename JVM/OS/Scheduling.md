## 스케줄(Scheduling)

OS가 프로세스들에 공정한 방향으로 CPU 자원을 할당하는 것. 

- **선점형 스케줄링**  
CPU 자원을 사용중인 프로세스가 있더라도, 우선순위에 따라 현재 CPU 자원을 점유 중인 프로세스를 준비 상태로 전환하고 강제로 다른 프로세스에 자원을 할당하는 경우

- **비선점형 스케줄링**  
현재 CPU 자원을 사용 중인 프로세스가 자원을 점유함. 해당 프로세스가 종료되기 전까지 다른 프로세스에 CPU를 할당할 수 없음. 



### 스케줄링 알고리즘 종류

- 선입 선처리 스케줄링(FCFS) : 비선점형 선입선처리 => 호위효과 발생 가능
- 최단 작업 우선 스케줄링(SJF) : 기본은 비선점형, 최단 작업 우선
- 라운드 로빈 스케줄링 : FCFS + **타임슬라이스** => 타임 슬라이스 단위가 너무 크면 호위효과 동일하게 발생
- 최소 잔여 시간 우선 스케줄링(SRT) : 최단 작업 우선 + 라운드 로빈 
- 우선순위 스케줄링 : 기아 현상 발생 가능 => 에이징 활용하여 해결 
- 다단계 큐 스케줄링 : 큐에도 우선순위를 부여하여 우선순위 대로 실행(큐와 큐 사이 이동 불가)
- 다단계 피드백 큐 스케줄링 : 큐 사이 이동이 가능하여 기아 문제 해결
